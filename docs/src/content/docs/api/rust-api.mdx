## Crate: `sml-core`

The Rust API is documented via `cargo doc`. Key types:

### Parsing

```rust
use sml_core::parse;

let doc = parse(r#"<skill interface="testing">Run tests</skill>"#).unwrap();
```

### Registry

```rust
use sml_core::SkillRegistry;

let mut registry = SkillRegistry::new();
registry.register_interface("testing".into(), Some("Run tests".into())).unwrap();
registry.register_implementation(
    "pytest-impl".into(),
    "testing".into(),
    Some("python".into()),
    None, None, 0,
).unwrap();
```

### Execution

```rust
use sml_core::{ExecutionContext, SkillRegistry, parse};

let registry = SkillRegistry::new();
let ctx = ExecutionContext::new(registry);
let doc = parse("some <skill ...>content</skill> text").unwrap();
let result = ctx.execute(&doc).unwrap();
```

### Key Types

| Type | Description |
|------|-------------|
| `Document` | Root AST node containing all parsed nodes |
| `Node` | Either `Skill { kind, params, children, span }` or `Text(String)` |
| `NodeKind` | `Invocation`, `InterfaceDefinition`, or `ImplementationDefinition` |
| `SkillRegistry` | Stores interfaces and implementations |
| `ExecutionContext` | Holds registry + handlers, executes documents |
| `Span` | Source location (`start`, `end` byte offsets) |

### Error Types

| Type | Description |
|------|-------------|
| `ParseError` | Syntax error with message and span |
| `RegistryError` | Duplicate names or orphan references |
| `ResolveError` | Resolution failures (ambiguous, not found, etc.) |
| `ExecutionError` | Runtime failures (handler error, retries exhausted) |

## Building Documentation

```bash
cargo doc --open -p sml-core
```
