---
title: Formal Grammar
description: EBNF grammar for the AML language.
---

## Overview

AML uses XML-like tags embedded in arbitrary text. The grammar defines skill nodes (three mutually exclusive types determined by the `define` attribute) and directive nodes (`<tool>`, `<session>`, `<agent>`).

## Node Types

```
Document       ::= (SkillNode | DirectiveNode | Text)*
SkillNode      ::= Invocation | InterfaceDefn | ImplDefn
DirectiveNode  ::= ToolDirective | SessionDirective | AgentDirective
Invocation     ::= '<skill' InvocationAttrs '>' Content '</skill>'
                 | '<skill' InvocationAttrs '/>'
InterfaceDefn  ::= '<skill' 'define="interface"' 'name=' QuotedStr InterfaceAttrs '>' InterfaceBody '</skill>'
InterfaceBody  ::= (Content | ParamDecl | ReturnsDecl | ReadsDecl | WritesDecl)*
ParamDecl      ::= '<param' ParamDeclAttrs '>' Content '</param>' | '<param' ParamDeclAttrs '/>'
ReturnsDecl    ::= '<returns' ReturnsDeclAttrs '>' Content '</returns>' | '<returns' ReturnsDeclAttrs '/>'
ReadsDecl      ::= '<reads>' Content '</reads>'
WritesDecl     ::= '<writes>' Content '</writes>'
ImplDefn       ::= '<skill' 'define="implementation"' 'name=' QuotedStr 'implements=' QuotedStr ImplAttrs '>' Content '</skill>'
ToolDirective  ::= '<tool' ToolAttrs '>' Content '</tool>' | '<tool' ToolAttrs '/>'
SessionDirective ::= '<session' SessionAttrs? '>' Content '</session>' | '<session' SessionAttrs? '/>'
AgentDirective ::= '<agent' AgentAttrs '>' Content '</agent>' | '<agent' AgentAttrs '/>'
```

## Attributes

```
InvocationAttrs ::= (InterfaceAttr | ImplAttr | NameAttr | LangAttr | FrameworkAttr | RetriesAttr | PolicyAttr | OnFailureAttr | TimeoutAttr)*
InterfaceAttrs  ::= DescriptionAttr?
ImplAttrs       ::= (LangAttr | FrameworkAttr | DescriptionAttr)*
ToolAttrs       ::= (NameAttr | AllowAttr | DenyAttr)+
SessionAttrs    ::= (NameAttr | IsolatedAttr)*
AgentAttrs      ::= NameAttr (ModelAttr | ModeAttr)*
ParamDeclAttrs  ::= NameAttr TypeAttr? RequiredAttr? DefaultAttr? ValuesAttr?
ReturnsDeclAttrs ::= NameAttr TypeAttr? ValuesAttr?
```

## Content Model

```
Content    ::= (SkillNode | DirectiveNode | ParamNode | Text)*
ParamNode  ::= '<param' 'name=' QuotedStr '>' ParamValue '</param>'
ParamValue ::= Text
Text       ::= (Char - '<')* | EscapedChar
EscapedChar::= '&lt;' | '&gt;' | '&amp;' | '&quot;' | '&apos;'
```

## Lexical Rules

- Attribute values are always quoted (single or double quotes)
- Whitespace between attributes is required
- Tag names are case-sensitive (`<skill>` only, not `<Skill>`)
- Comments (`<!-- ... -->`) are stripped during parsing
